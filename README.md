# 🔐 React Route Authentication (Using Cookies)

## 📌 Overview

**Route Authentication** is the process of verifying user credentials before allowing access to a specific route/path in a React application.

It ensures that only authorized users can access protected pages like dashboards, profiles, etc.

---

## 🔑 Key Concepts

### ✅ Authentication
- Process of **verifying user identity**
- Example:
  - User ID & Password
  - Security Tokens
  - Biometrics (Fingerprint, Face ID)

---

### 🔒 Authorization
- Process of **restricting access to resources**
- Example:
  - Admin vs User access
  - Protected routes

---

## ⚙️ Route Authentication Using Cookies

Cookies are used to **store user session data** and manage authentication state.
